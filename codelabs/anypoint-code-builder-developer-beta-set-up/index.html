Overview

Welcome to the Anypoint Code Builder Beta walkthrough! Anypoint Code Builder is MuleSoft's next generation IDE to design, develop and deploy APIs and integrations. The IDE is developed on top of one of the most used IDEs among developers, VS Code. The Anypoint Code Builder Beta is now available to those participating in the Beta program.

In this section, we will set up Anypoint Code Builder for the following codelabs.

What you'll learn

  • How to set up the Anypoint Code Builder docker image.
  • Review its look and feel.

What you'll need

  • Docker - you can install it from here
  • Anypoint Platform Account
  • Sign up to the Anypoint Code Builder Beta program - contact our product management director Simone Geib

In this section, we'll download the docker image for Anypoint Code Builder and load it on your environment.

Set up a folder/directory to manage the Anypoint Code Builder image.

During this Beta program, several versions will be expected to be released. For the sake of manageability, we recommend you set up a folder/directory dedicated to the version associated to the image you are going to download in the next step.

For example:

$ mkdir -p $HOME/acb/images/<version number>

Download the image from here.

Go to the directory where downloaded image is located

Use cd command to change the current directory.

Install the image onto your machine

Run the command below to unzip the image and load it to your docker (Note: directory and the version may differ if you download the newer image.):

$ gunzip -c ~/acb/images/23.1.3/mule-dx-dist-v23.1.3.tar.gz | docker load

When the installation is successfully completed, you can verify it on your docker desktop like below:

Open up a terminal if it's not already open

Start the image

Run the command below to run the docker image you installed in the previous section. It will automatically open up in your browser using the following URL: http://localhost:8080

$ docker run -i --rm --name mule-dx -d -p8080-8100:8080-8100 -v $HOME:$HOME artifacts.msap.io/mulesoft/mule-dx-dist:v23.1.3 && sleep 3 && open http://localhost:8080

You may see a blank page initially while the docker image is starting up.

Make sure the image is up and running
